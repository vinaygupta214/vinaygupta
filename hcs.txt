                                                                           HTML CHEAT SHEET :->


<html>           </html>  - The HTML TAG Specifies an html document.
<base>      -     The BASE TAG Specifies URL which non-absolute URLs are relative to.
<head>           </head> - The  HEAD TAG provides general information (metadata) about the document, including its title and links to its scripts and style sheets.
<link>      -     The LINK TAG specifies relationships between the current document and an external resource.
<meta>      -    The META TAG represents any metadata information.
<style>          </style> - The STYLE TAG instructions written inside that element are expected to be CSS.
<title>          </title> - The TITLE TAG defines the title of the document, shown in a browser's title bar or on the page's tab. 
<address>        </address> - The ADDRESS TAG supplies contact information for its nearest <article> or <body> ancestor; in the latter case, it applies to the whole document.
<article>        </article> - Section of the page content, such as a blog or forum post.
<aside>          </aside> - Content related to surrounding elements that doesn't belong inline,such as a advertising or quotes. Which could be considered separate from that content.
<body>           </body> -  Main content of the document.
<header>         </header> - Navigation or introductory elements for the current section and like a logo, wrapped section's header, a search form, and so on.
<footer>         </footer> - A footer typically contains information about the author of the section, copyright data or links to related documents.
<h1></h1>  TO    <h6></h6> - Heading for the current section.
<hgroup>         </hgroup> - It defines a single title that participates in the outline of the document as the heading of the implicit or explicit section that it belongs to.
<nav>            </nav> - A section of a page that links to other pages.
<section>        </section> - It represents a generic section of a document, i.e., a thematic grouping of content, typically with a heading.
<blockquote>     </blockquote> - Quote from another source.A URL for the source of the quotation may be given using the cite attribute.
<br>     -       The Break Tag using for Line Break.
<dd>             </dd> - The DD Tag (HTML Description,difinition Element) indicates the description of a term in a description list (<dl>) element.
<div>            </div> - Container(using the class or id attributes) or section with no semantict (such as <article> or <nav>) meaning.
<dl>             </dl> - An association list consisting of zero or more name-value groups (a Description List).
<dt>             </dt> - (Discription Term) Term, or name, part of a term-description group in a description list.
<figcaption>     </figcaption> - (Figcaption Tag) Caption or legend for the figure element.
<figure>         </figure> - The Figure element represents self-contained content, frequently with a caption (<figcaption>), and is typically referenced as a single unit.
<hr>       -     It represented a horizontal rule. It may still be displayed as a horizontal rule in visual browsers.
<li>             </li> - List item.
<main>           </main> - Specifies the main content area of an HTML document.
<ol>             </ol> - Typically, ordered-list items are displayed with a preceding numbering, which can be of any form, like numerals, letters or Romans numerals or even simple bullets.
<ul>             </ul> -  Typically, unordered-list items are displayed with a bullet, which can be of several forms, like a dot, a circle or a squared.
<p>              </p> - Paragraph content.
<pre>            </pre> - A block of preformatted text.
<a>              </a> - Anchor Tag defines a hyperlink to a location on the same page or any other page on the Web.
<abbr>           </abbr> - Abbreviation or acronym represents an abbreviation and optionally provides a full description for it.
<b>              </b> - Bold
<i>              </i> -Italic
<u>              </u> -Underline
<bdi>            </bdi> - Defines directional formatting for content. The HTML <bdi> Element (or Bi-Directional Isolation ) isolates a span of text that might be formatted in a different
                          direction from other text outside it.                 
<bdo>            </bdo> - Bidirectional override is used to override the current directionality of text. It causes the directionality of the characters to be ignored in favor of the 
                          specified directionality.                                   
<cite>     -     It must include the title of a work or a URL reference, which may be in an abbreviated form according to the conventions used for the addition of citation metadata.
<code>           </code> - It represents a fragment of computer code. By default, it is displayed in the browser's default monospace font.
 <del>           </del> - Text that has been removed during document editing. 
<dfn>            </dfn> - Defining instance of a term. 
<em>             </em> - Text that should be emphasized.
<ins>            </ins> - Text that has been inserted during document editing.
<kbd>            </kbd> - The HTML Keyboard Input Element represents user input and produces an inline element displayed in the browser's default monospace font.
<mark>           </mark> - Text highlighted for referencing elsewhere.
<q>              </q> - Phrasing content quoted from another source.
<rp>             </rp> - Contains semantically meaningless markup for browsers that don't understand ruby annotations. 
<rt>             </rt> - The HTML <rt> Element embraces pronunciation of characters presented in a ruby annotations, which are used to describe the pronunciation of East Asian characters.
<ruby>           </ruby> - Contains text with annotations, such as pronunciation hints.
<s>              </s> - Strikethrough text that is outdated or no longer accurate.
<samp>           </samp> - Sample output of a program.
<small>          </small> - Small text, such as fine print.
<span>           </span> - Container with no semantic meaning. The HTML <span> element is a generic inline container for phrasing content, which does not   inherently represent anything.
<strong>         </strong> - Text that is important. The HTML Strong Element (<strong>) gives text strong importance, and is typically displayed in bold.
<sub>            </sub> - Subscript defines a span of text that should be displayed, for typographic reasons, lower, and often smaller, than the main span of text.
<sup>            </sup> - Superscript defines a span of text that should be displayed, for typographic reasons, higher, and often smaller, than the main span of text.
<time>           </time> - Time defined in a machine readable format. The HTML <time> element represents either a time on a 24-hour clock or a precise date in the Gregorian calendar (with
                           optional time and timezone information).
<var>            </var> -Variable represents a variable in a mathematical expression or a programming context.
<wbr>            </wbr> - Word Break opportunity represents a position within text where the browser may optionally break a line, though its line-breaking rules would not otherwise create
                          a break at that location. 
<caption>        </caption> - Title of a table.
<table>          </table> - Table of multi-dimensional data. The Table represents tabular data: information expressed via two dimensions or more.
<tbody>          </tbody> -Contains rows that hold the table's data.
<td>             </td> - Table cell. The Table cell defines a cell of a table that contains data. It participates in the table model.
<tfoot>          </tfoot> - Contains rows with summary of data.
<th>             </th> - Table heading. 
<thead>          </thead> - Contains rows with table headings.
<tr>             </tr> - A row of cells in a table. The table row defines a row of cells in a table. Those can be a mix of <td> and <th> elements.
<button>         </button> - A button. The HTML <button> Element represents a clickable button.
<data>           </data> - Allows for machine-readable data to be provided.
<datalist>       </datalist> - Define sets of options.
<fieldset>       </fieldset> - Set of form controls grouped by theme.
<legend>         </legend> - Define a name for a fieldset.
<form>           </form> - Used to create an HTML form for user input.
<input>   -      Generic form input.It is used to create interactive controls for web-based forms in order to accept data from the user.
<label>          </label> -Caption for a form control.
<meter>          </meter> - Control for entering a numeric value in a known range.
<optgroup>       </optgroup> - Group of option. In a Web form, the HTML <optgroup> elementmcreates a grouping of options within a <select> element.
<option>         </option> - Single option within a select control.
<output>         </output> - Contains the results of a calculation. 
<progress>       </progress> - Control for displaying progress of a task.
<select>         </select> - Control for selecting from multiple options.
<textarea>       </textarea> - Multiline free-form text input.
<embed>    -     Integration point for an external (typically non-HTML) application orinteractive content.
<iframe>         </iframe> - Nested browser frame. The iframk represents a nested browsing context, e†‐ectively embedding
another HTML page into the current page. 
<img> -  It represents an image in the
document.
<object> </object> - External resource such as an image, iframe or plugin.
<param> - Parameters for the parent object. 
<source> </sourse> - Alternative sources for parent video or audio elements. 
<area>  - Hyperlink with some text and a corresponding area on an image
map, or a dead area on an image map.
<audio> </audio> - Sound or audio stream. 
<map> </map> - Image map for adding hyperlinks to parts of an image.
<track> - Specifies external timing track for media element.
<video> </video> - Used for playing videos or movies.
<details> </details> - Contains additional information, such as the contents of an
accordian view. 
<dialog> </dialog> - It represents a dialog box or other
interactive component, such as an inspector or window.
<menu> </menu> - Set of commands. It represents a group of
commands that a user can perform or activate.
<summary> </summary> - Caption of a details element.
<canvas> <canvas> - Bitmap which is editable by client side scripts.
<noscript> </noscript> - Contains elements that are part of the document only if scripting is
disabled.
<script> </script> - Inline or linked client side scripts.
